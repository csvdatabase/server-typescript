# Security Policy

## Supported Versions

Security fixes are provided for the newest stable server release. Users should
update to the latest patched image before reporting a resolved vulnerability.

## Reporting a Vulnerability

Do not open a public issue for an unpatched vulnerability. Use GitHub's private
vulnerability reporting from this repository's **Security** tab. If it is not
available, contact a repository maintainer through a private channel.

Include the affected version or image digest, impact, reproduction steps, and any
suggested mitigation. Do not include real credentials or production data.
